Overview

This app helps English learners memorize vocabulary words and short phrases, including spelling, English meaning, Chinese meaning, IPA American pronunciation marks when available, and audio pronunciation. It can instantly translate an English word or short phrase to Chinese meaning, can use Chinese input to find likely English terms, and can save user-specific vocabulary and quiz progress. It also provides a dedicated spelling-practice track (a separate spelling list reviewed by listening and typing) and optional online AI-assisted word details. Each requirement is marked with a unique ID so it can be tracked clearly.

Product transition note, 2026-05-25: WordLover is now treated as an early product build, not only a feasibility prototype. iPhone is the primary use case and Windows is the automation/stress-test fallback. Android remains in scope long term but is deferred until the end. The Windows PWA dictionary validation passed with the current generated SQLite dictionary, and the real iPhone 17 Pro validation was reported to work well with fast web-app startup and fast dictionary loading while online. Automated iPhone reports later verified service worker readiness, IndexedDB dictionary persistence, OPFS persistence, encrypted export/import, mock sync, and lookup p95 far under 1 second after the dictionary is open. A later iPhone offline test found that the app shell opens without Wi-Fi, but the original build could not load/search the dictionary offline until the dictionary was persisted locally. The app now saves the dictionary to durable browser storage and falls back to that copy offline; this passed on Windows fallback automation and the iPhone suite confirms persistence. The current `sql.js` query engine is not sufficient for the production iPhone DRAM target because it loads the full dictionary into JS/WASM memory. Remaining product validation should focus on true Wi-Fi-off Home Screen dictionary search confirmation, real Google Drive OAuth sync, measured iPhone memory behavior, and a production dictionary engine such as `wa-sqlite`+OPFS or a sharded dictionary package.

Status audit note, 2026-05-26: the Status column is now the single source of truth for current implementation state against the product requirements. Status values are `done` for implemented in the current PWA product build, `partial` for implemented only partly, config-gated, validation-gated, or missing required sub-behavior, `open` for not implemented, and `deferred` for intentionally postponed work such as Android validation.

Status audit note, 2026-05-29 (verified against code): The product was rebranded to WordFan (user-facing strings and the web manifest only; internal data identifiers, IndexedDB names, snapshot app id, and the Drive file name are unchanged for data continuity) and is served over HTTPS from the custom domain wordfan.app via GitHub Pages. The production Google OAuth client ID is now bundled in `wordlover-config.js` so fresh installs do not prompt for one (a per-device override remains). The shipped dictionary is the slim ~32 MB / ~100k-entry SQLite set; it auto-installs on the first online launch (the manual load button is only an offline fallback) and is fetched in a single integrity-checked GET (byte length + SHA-256 vs manifest) that self-heals a corrupt local copy. The former dwell-based autosave (reqs 32-36, 42) was intentionally replaced by an explicit "Save on Return" setting (vocabulary [default], spelling, both, or none); `scheduleAutosave` is now a no-op. A full Spelling practice track was added. Chinese→English translation lookup with English synonym candidates is confirmed working. The search input now accepts letters, digits, spaces, hyphens, apostrophes, and Chinese characters and drops other stray characters. New requirements 191-215 track these additions; the autosave rows below were updated to the Save-on-Return model.
